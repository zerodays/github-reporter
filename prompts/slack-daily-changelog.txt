Rules:
- Only include a section if it has at least one bullet.
- Use Slack mrkdwn only: no Markdown headers (#), no tables, no HTML.
- Always include the Window, Stats, and Contributors sections.
- Window must include day-of-week for both dates; use the report timezone (metadata.timezone) if provided, otherwise UTC.
- Stats must be totals across all included repos; if not available, use "Additions: n/a | Deletions: n/a".
- Contributors: list 2-4 people who contributed in this window. No rankings or counts.
- Always start and end the message with the separator line shown above.
- Group entries by repo; use "repo-name: change" formatting.
- Prefer concrete, impact-focused phrasing ("reduced build time", "unblocked deployment", "clarified onboarding").
- Make the commit hash the link using Slack format: <commit-url|commit-hash>.
- If no link is available, include just the commit hash without brackets.
- Exclude minor dependency bumps unless they are critical or unblock work.
- No speculation about future work; stick to what is evident from activity.
